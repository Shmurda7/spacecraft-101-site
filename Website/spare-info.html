Intel released the 8086 processor in 1978, launching the x86 architecture. The 8086 was a 16-bit CISC processor with 29,000 transistors running at 5-10 MHz. The architecture was later extended to 32-bit (IA-32) with the 80386 in 1985 and to 64-bit (x86-64) by AMD in 2003. Today x86 dominates desktop, server, and many embedded applications, though its power consumption is a concern for spacecraft.

The RISC philosophy originated at UC Berkeley in 1980 under David Patterson's RISC-I project. ARM (Acorn RISC Machine) began as a collaboration between Acorn Computers and VLSI Technology in 1985 — the ARM1 had just 25,000 transistors (roughly the same as the 8086) but consumed only 0.1 W. Today ARM powers billions of mobile and embedded devices. RISC-V, an open-standard ISA, started at UC Berkeley in 2010 and is gaining traction in space applications for its openness and customizability.

PowerPC

DRAM (Dynamic RAM)

DRAM uses a capacitor-transistor cell (1T1C) that stores data as an electrical charge. Because capacitors leak charge over time, DRAM requires periodic refresh to retain data. It is the densest and most cost-effective volatile memory technology, making it the standard choice for main system memory.

FeatureDRAM (Dynamic RAM)
Storage CellCapacitor + transistor (1T1C — one transistor, one capacitor)
Data RetentionRequires periodic refresh (~64 ms interval)
SpeedModerate (~50–100 ns access time)
DensityVery high — single transistor per cell enables multi-Gb chips
CostLow per bit — cheapest volatile memory technology
PowerHigher — needs refresh circuitry; standby power still significant
Typical UseMain system memory (RAM modules, DDR4/DDR5)
Data RemanenceLimited — residual charge persists seconds to minutes after power-off; longer at low temperatures

DRAM was invented by Robert Dennard at IBM in 1966. Modern DDR5 DIMMs pack 16–64 GB per module with transfer rates exceeding 60 GB/s. In spacecraft, DRAM is often derated (run below max speed) and paired with ECC to mitigate SEUs. Data remanence at low temperatures is a security concern — sensitive data can persist longer than expected after power-off.

SRAM (Static RAM)

SRAM uses a flip-flop circuit (typically 6 transistors per cell) that holds data as long as power is supplied — no refresh needed. It is faster and more reliable than DRAM but significantly less dense and more expensive, limiting it to applications where speed matters most.

FeatureSRAM (Static RAM)
Storage CellFlip-flop (6 transistors per bit — 6T cell)
Data RetentionHolds data as long as power is on; no refresh required
SpeedFast (~1–10 ns access time)
DensityLow — 6 transistors per cell limits capacity
CostHigh per bit — 4–6× more expensive than DRAM
PowerLower dynamic power than DRAM (no refresh); higher static leakage
Typical UseCPU cache (L1, L2, L3), FPGA block RAM, register files
Data RemanenceMinimal — loses data within microseconds when powered off

SRAM predates DRAM and was used in early magnetic core memory systems. It remains essential as the cache hierarchy that bridges the speed gap between the CPU and main memory. In FPGAs, distributed SRAM (LUTRAM) and block RAM (BRAM) provide fast on-chip storage for buffers, FIFOs, and small lookup tables. Space-grade SRAM devices often include built-in ECC and latch-up protection.

VRAM (Video RAM)

VRAM is a dual-ported variant of DRAM designed specifically for graphics processing. Its defining feature is the ability to perform simultaneous reads (for display refresh) and writes (from the GPU), eliminating the contention that would occur with single-ported memory in a graphics pipeline.

FeatureVRAM (Video RAM)
Storage CellDual-ported DRAM — two independent access paths per cell
Data RetentionRequires periodic refresh (same as DRAM)
SpeedFast (~2–10 ns per port depending on generation)
DensityModerate — dual-port structure increases cell size over DRAM
CostHigher than DRAM due to dual-ported architecture
PowerModerate to high — dual-port operation increases switching activity
Typical UseGraphics card frame buffer, GPU texture memory
Key FeatureDual-ported — read (display) and write (render) occur simultaneously

VRAM was introduced in the 1980s as a specialized memory for high-resolution graphics. Modern GPUs have largely moved to GDDR (Graphics DDR) — a high-bandwidth variant of DDR SDRAM — and HBM (High Bandwidth Memory), which stacks DRAM dies vertically with through-silicon vias (TSVs). While not strictly VRAM, these technologies serve the same purpose. In spacecraft, radiation-tolerant GPUs with dedicated VRAM may be used for image processing and compression.

FRAM (Ferroelectric RAM)

FRAM uses a ferroelectric crystal layer (typically PZT — lead zirconate titanate) between two electrodes. Data is stored in the polarization state of the crystal, which remains intact when power is removed. FRAM combines the speed of RAM with non-volatile storage, but is limited in density and capacity.

FeatureFRAM (Ferroelectric RAM)
Storage MechanismFerroelectric capacitor (PZT crystal) — polarization stores data
Data RetentionNon-volatile — retains data for 10+ years without power
SpeedFast (~50–100 ns read/write, comparable to DRAM)
DensityLow — larger cell size limits capacity to ~8 Mb per chip
CostHigher per bit than DRAM or Flash
PowerVery low — no refresh, non-volatile, low active power
EnduranceVery high (10¹⁰ – 10¹⁵ write cycles)
Typical UseEmbedded systems, smart cards, RFID, spacecraft configuration storage
Radiation ToleranceGood — not affected by magnetic interference; moderate SEU immunity

FRAM was first developed by Ramtron (now part of Cypress/Infineon) in the 1990s. It offers the fastest write speed of any non-volatile memory technology and consumes significantly less power than EEPROM or Flash. In spacecraft, FRAM is used for critical configuration data, telemetry counters, and black-box event logging where instant non-volatile writes are essential. Fujitsu and Infineon are the primary suppliers of FRAM devices.

MRAM (Magnetoresistive RAM)

MRAM stores data using magnetic tunnel junctions (MTJs) — two ferromagnetic plates separated by a thin insulating barrier. Data is represented by the magnetic orientation of the free layer relative to the fixed layer. MRAM combines near-SRAM performance with non-volatility and excellent radiation tolerance.

FeatureMRAM (Magnetoresistive RAM)
Storage MechanismMagnetic tunnel junction (MTJ) — ferromagnetic plates + tunneling barrier
Data RetentionNon-volatile — retains data for 20+ years without power
SpeedFast (~10–50 ns read/write, approaching SRAM)
DensityModerate — 1–256 Mb per chip (STT-MRAM scaling to Gb)
CostHigher per bit than DRAM; complex manufacturing process
PowerModerate — no refresh; write current is the main draw
EnduranceVery high (10¹⁵+ cycles — essentially unlimited)
Typical UseHigh-speed persistent memory, aerospace storage, cache replacement, buffer memory
Radiation ToleranceExcellent — immune to SEUs; no charge-based storage mechanism

MRAM was first commercialized by Freescale (now NXP) in 2006. The newer STT-MRAM (Spin-Transfer Torque MRAM) variant reduces write current and improves density, making it a candidate for universal memory — replacing both DRAM and NAND Flash in some applications. Everspin is the leading commercial supplier, and NASA/ESA are evaluating MRAM for next-generation spacecraft mass memory due to its radiation hardness and high endurance.

Key Differences Between Generations

  • Voltage — each generation drops voltage significantly (2.5V → ~1.0V), reducing power consumption
  • Prefetch buffer — doubled from 2n to 16n, allowing more data per memory access
  • Banks — increased from 4 to 32+, enabling more parallel accesses
  • Bandwidth — DDR1 peaks at 3.2 GB/s (PC-3200), DDR5 exceeds 60+ GB/s
  • DDR5 introduced on-die ECC, dual 32-bit sub-channels per DIMM, and an on-module PMIC (power management IC)
  • DDR6 (in development) is expected to use PAM-4 signaling (4-level pulse amplitude modulation) to achieve even higher data rates without doubling pin speed

In spacecraft, error correcting code (ECC) memory is critical — ionizing radiation can flip individual bits (single-event upsets (SEUs)). Many rad-hard memory devices include built-in ECC or utilize scrubbing techniques where memory is periodically read, corrected, and rewritten to prevent error accumulation.

SDRAM (Synchronous DRAM) was first standardized by JEDEC in 1993. DDR SDRAM doubled the transfer rate without increasing the clock by transferring data on both edges of the clock signal. Each generation roughly doubles bandwidth while cutting voltage — DDR1 modules peaked at 3.2 GB/s using 2.5V, while DDR5 modules deliver 60+ GB/s at just 1.1V. For spacecraft, DDR memory is often derated (run below max speed) and paired with ECC to mitigate radiation-induced single-event upsets (SEUs).

Intel introduced NOR flash in 1988 as a high-reliability, random-access memory technology. Toshiba's Fujio Masuoka invented NAND flash the following year (1989), prioritizing density and cost over random access. Both were breakthroughs over EEPROM — Fujio Masuoka also invented the original floating-gate EEPROM technology that made flash possible.

NOR vs NAND Flash

Two memory architectures for storing information in memory cells made from floating-gate transistors.

ArchitectureEach memory cell connects directly to bit line (resembles NOR gate)Memory cells connected in series (resembles NAND gate)
Read / AccessRandom access, short read times. Best for code execution (XIP — eXecute In Place).Slower read, indirect I/O-mapped interface. Requires shadowing to RAM for code execution.

Cost / DensityLarger cell size, higher cost per bit, slower write/eraseSmaller cell size, lower cost per bit, high write/erase speed
Memory Range64 Mb – 2 Gb1 Gb – 16 Gb (better for data storage)
Erase Block64 KB – 256 KB8 KB – 32 KB
Erase Speed~520 ms per 128 KB block~3.5 ms per 128 KB block (~150x faster)

Note that often there is NOR or NAND flash memery defined by the communication protocol used to interface with hardware. For example SPI NOR is a type of flash memory that uses the SPI communication protocol.

The first FPGA was invented by Ross Freeman at Xilinx in 1985. The XC2064 had just 64 logic blocks and 58 I/O pins using 2 µm technology — a far cry from modern FPGAs with millions of logic cells. FPGAs are widely used in spacecraft because they can be reconfigured after launch, allowing bug fixes and feature upgrades without a hardware respin. NASA's James Webb Space Telescope uses radiation-tolerant FPGAs for instrument control and data processing.
